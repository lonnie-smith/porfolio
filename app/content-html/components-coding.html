One challenge has been configuration: how do we ensure a component is configurable enough to cover our constantly-evolving use cases without sacrificing ease-of-use, reliability, or consistency?

Usually, what works is to add new configuration properties to a widget so that modifying the appearance of the widget is as easy as adding a new attribute to the HTML. Here are full code samples for the multiple-choice widget. Wherever I can, I like to use Sass and Coffeescript. I always try to keep stylesheets, scripts, and markup in separate files, and I keep each component’s source in separate folders. Aside from JSDoc-style comments that generate the API documentation, I try to keep comments pretty minimal. Grunt automates compiling and concatenating source files; Bower manages dependencies (and for deploying the library to client code), and we host the library in a private GitHub repo.
